# Shared Data and Resource Setup

Run this once before any tier. It builds the synthetic AkzoNobel coatings data and the resources every notebook and agent depends on. There are two ways to set up, and you can mix them: a data load that runs from your laptop, and programmatic creation of the Genie spaces and the Knowledge Assistant backbone.

> Note: all data is synthetic. Product names, accounts, suppliers, and documents are invented for the workshop.

---

## What gets created

| Resource | Where | Built by |
|---|---|---|
| Finance, SCM, Commercial tables | `<catalog>.akzo_finance` / `akzo_scm` / `akzo_commercial` | generators + `load_to_uc.py` |
| Document volume (safety sheets, contracts) | `/Volumes/<catalog>/akzo_docs/raw` | `load_to_uc.py` |
| Three Genie spaces (Finance, SCM, Commercial) | Genie | `setup/setup_genie_spaces.py` |
| Vector Search endpoint and index | `akzo_workshop_vs`, `akzo_docs.chunks_idx` | `setup/setup_vector_search.py` |

Nothing about a workspace is baked in. Configure with arguments or environment variables. The only required value is your catalog.

```
--catalog / AKZO_CATALOG           required; your Unity Catalog (on a Vocareum lab, your assigned catalog)
--profile / DATABRICKS_CONFIG_PROFILE  optional; omit to use ambient auth (the default when running inside the workspace)
--warehouse / DATABRICKS_WAREHOUSE_ID  optional; the first SQL warehouse in the workspace is used if unset
AKZO_VS_ENDPOINT                   optional; Vector Search endpoint name (default: akzo_workshop_vs)
```

On Vocareum you run inside the workspace, so the profile is not needed and the warehouse is discovered. You only set your catalog.

## Step 1. Generate and load the structured data

The generators produce deterministic parquet under `data/output/`, then `load_to_uc.py` creates the schemas, a staging volume, and the tables. Both steps are idempotent, so re-running is safe.

```bash
pip install pandas numpy pyarrow
python data/generate_finance.py
python data/generate_scm.py
python data/generate_commercial.py

# loads parquet into Unity Catalog using the authed Databricks CLI
AKZO_CATALOG=<catalog> python data/load_to_uc.py
```

The documents (safety data sheets and contracts) are generated by `data/generate_docs.py`, which needs `reportlab`. `load_to_uc.py` uploads them to the `akzo_docs` volume if they are present.

This was run for this workshop and all thirteen tables loaded, with non zero row counts in every domain.

---

## Step 2. Create the Genie spaces from code

Rather than clicking through the UI, create the three domain Genie spaces programmatically with the Genie Spaces API. Each space is grounded with table descriptions, one instruction block, example SQL, and sample questions. The script is idempotent: a space whose title already exists is replaced.

```bash
pip install databricks-sdk
python data/setup/setup_genie_spaces.py --catalog <catalog>
```

It prints the three space ids and writes them to `data/setup/space_ids.json`, which the L300 supervisor reads to call the real Genie Conversation API. This was run and verified: each space answers natural language questions and returns correct SQL plus results over the coatings tables.

---

## Step 3. Create the Knowledge Assistant backbone from code

The Knowledge Assistant answers document questions over a Vector Search index. This script ensures the endpoint and the delta sync index exist and reports when the index is ready.

```bash
pip install databricks-sdk
python data/setup/setup_vector_search.py --catalog <catalog>
```

This was run and verified: the index reports `ready=True`. Point a Knowledge Assistant at the endpoint and index it prints.

## Folder contents

| Path | Purpose |
|---|---|
| `generate_finance.py` `generate_scm.py` `generate_commercial.py` | Generate the structured domain tables as parquet |
| `generate_docs.py` | Generate the safety sheet and contract PDFs |
| `load_to_uc.py` | Load parquet and documents into Unity Catalog (idempotent) |
| `setup/setup_genie_spaces.py` | Create the three Genie spaces from code |
| `setup/setup_vector_search.py` | Create the Vector Search endpoint and index from code |
| `setup/query_genie.py` | Quick check that a Genie space answers a question |
| `setup/finance_space.md` `setup/scm_space.md` `setup/commercial_space.md` | Human readable specs for each Genie space |
| `output/` | Generated parquet (not committed) |
